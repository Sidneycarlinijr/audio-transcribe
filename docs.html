Scripts Disponiveis

transcribe_audio.py Principal

Script completo: divide audios longos, transcreve cada segmento via Gemini, junta tudo e salva como Markdown com tema automatico.

python transcribe_audio.py audio.mp3 python transcribe_audio.py --folder ~/Audios python transcribe_audio.py audio.mp3 --segment-duration 5 python transcribe_audio.py audio.mp3 --output ~/Transcricoes

Saida: ~/Documentos/Transcricoes/2026-04-02 — Tema.md

split_audio.py Utilitario

Apenas divide audios longos em segmentos menores (sem transcricao). Util para pre-processar antes de transcrever ou para enviar partes menores manualmente.

python split_audio.py audio.mp3 python split_audio.py --folder ~/Audios python split_audio.py audio.mp3 --segment-duration 15 python split_audio.py audio.mp3 --output ~/Partes

Saida: audio_partes/audio_parte_01.mp3, audio_parte_02.mp3, ...

transcribe_segment.py Leve

Transcreve um unico arquivo de audio. Inclui retry com backoff exponencial para rate limiting. Saida via stdout — ideal para pipelines e composicao com outros scripts.

python transcribe_segment.py audio.mp3 > transcricao.txt

Retry: 3 tentativas, backoff de 40s * tentativa

audio-transcription-queue.gs Cloud

Versao Google Apps Script para processar audios diretamente do Google Drive. Sistema de fila persistente com upload resumable (chunks de 8MB), estado recuperavel e saida em Google Docs.

Quando usar a versao cloud?

Quando voce quer processar audios automaticamente sem rodar nada local. Basta jogar o audio numa pasta do Drive e o trigger processa sozinho.

Limitacao: Timeout de 6 minutos por execucao (contornado com fila + estado persistente).

Comparacao: Local vs Cloud

Aspecto Python (Local) Apps Script (Cloud)
Timeout Sem limite 6 min/execucao (fila contorna)
Tamanho Audio Qualquer Qualquer (upload resumable)
Onde Roda Seu computador Google Cloud
Automatico Manual (CLI) Sim (trigger a cada 5-10 min)
Saida Markdown local Google Docs no Drive
Dependencias Python, ffmpeg, pydub Nenhuma (roda no Google)
Retry Sim (transcribe_segment.py) Sim (estado persistente)

Setup: Python (Local)

Criar venv e instalar dependencias

python3 -m venv ~/venvs/transcribe source ~/venvs/transcribe/bin/activate pip install google-generativeai pydub

Instalar ffmpeg

sudo apt install ffmpeg

Configurar API Key

Gere sua key em aistudio.google.com/apikey

# Adicione ao ~/.zshrc ou ~/.bashrc export GEMINI_API_KEY="sua_key_aqui"

Adicionar alias (opcional)

# Evita ativar venv toda vez alias transcribe="~/venvs/transcribe/bin/python transcribe_audio.py"

Setup: Google Apps Script (Cloud)

Criar projeto no Apps Script

Va em script.google.com e crie um novo projeto. Cole o conteudo de audio-transcription-queue.gs.

Ativar Drive API

No editor, va em Servicos (+) e adicione Drive API v3.

Configurar Propriedades do Script

Va em Configuracoes do Projeto > Propriedades do script e adicione:

Propriedade Valor
GEMINI_API_KEY Sua key do Google AI Studio
FOLDER_ID ID da pasta do Drive com os audios
OUTPUT_FOLDER_ID ID da pasta do Drive para saida

Inicializar e criar trigger

// Execute uma vez: inicializarSistema() // Depois configure trigger: // Triggers > Add trigger > processarProximoDaFila > Every 5 minutes // Para adicionar audios manualmente: enfileirarAudios()
Funcoes de diagnostico

verStatusFila() — mostra estado da fila e processamento atual

limparFilaCompleta() — reseta fila e estado

forcarReprocessamento(fileId) — reprocessa um arquivo especifico

Formatos Suportados

.mp3 .m4a .wav .ogg .aac .flac

O formato .m4a e o usado pelo Plaud e outros gravadores de voz.

Seguranca

🔐

API Keys via variavel de ambiente — Nenhuma chave hardcoded no codigo. Scripts Python leem de GEMINI_API_KEY (env var). Apps Script le de Propriedades do Script.

📁

.gitignore configurado — Arquivos .env, audios e venvs sao ignorados pelo git.

🚫

Sem dados sensiveis — Folder IDs do Drive e output paths sao configurados externamente, nunca no codigo.

Antes de compartilhar

Verifique que seu ~/.zshrc ou ~/.bashrc nao esta commitado — ele contem a GEMINI_API_KEY.

Troubleshooting

Erro: GEMINI_API_KEY nao definida

Configure a variavel de ambiente:

export GEMINI_API_KEY="sua_key"

Erro: pydub nao instalado

Ative o venv e instale:

source ~/venvs/transcribe/bin/activate pip install pydub google-generativeai

Erro: ffmpeg nao encontrado

sudo apt install ffmpeg

Rate limit / 429 Error

transcribe_segment.py ja tem retry automatico. Para transcribe_audio.py, reduza segmentos:

python transcribe_audio.py audio.mp3 --segment-duration 5

Apps Script: Timeout / Estado travado

O sistema de fila retoma automaticamente. Se travar, use:

// Ver estado atual verStatusFila() // Se travou, limpar e recomecar limparFilaCompleta() enfileirarAudios()
